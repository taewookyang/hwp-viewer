# 배포 방법

## 권장 배포 방식
- 정적 호스팅 사용: GitHub Pages, Netlify, Cloudflare Pages 모두 가능
- 이 앱은 `index.html`을 파일앱에서 직접 여는 방식보다 **HTTP/HTTPS 주소로 여는 방식**이 안정적입니다.

## GitHub Pages 업로드 순서
1. 새 저장소 생성
2. 이 폴더의 파일들을 저장소 루트에 업로드
3. `Settings → Pages`
4. Source를 배포 브랜치로 선택
5. 몇 분 뒤 생성된 URL 접속
6. 휴대폰 Chrome에서 `홈 화면에 추가`

## 포함 파일
- `index.html`
- `styles.css`
- `app.js`
- `rhwp.js`
- `rhwp_bg.wasm`
- `manifest.json`
- `sw.js`
- `icon.svg`

## 개인정보/보안 주의
- 이 앱은 **문서 파일 자체를 서버에 업로드하지 않습니다.**
- 문서 내용은 브라우저 안에서 처리되도록 설계했습니다.
- 다만, 호스팅 서비스 접속 자체는 일반 웹 접속이므로 서비스 제공자 쪽 접속 로그는 남을 수 있습니다.
- 민감한 문서는 가능한 한 신뢰 가능한 개인 호스팅 또는 제한된 접근 환경에서 사용하세요.
