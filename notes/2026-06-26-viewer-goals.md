# HWP Viewer 개선 목표

**목표:** 안드로이드에서 링크만으로 안정적으로 열리고, 문서를 빠르게 찾아보고 이동할 수 있는 실사용형 HWP/HWPX 뷰어로 개선한다.

## 1차 목표 (지금 우선)
1. 초기 진입 안정화
   - 빈 화면/무한 로딩/버튼 비활성 같은 진입 버그 제거
2. 문서 열기 안정화
   - 작은 문서는 확실히 열리게 하고, 실패 시 명확한 오류 표시
3. 확대/축소 버튼 추가
   - 핀치 없이도 확대/축소/원래 크기 가능
4. 문서 정보 표시
   - 파일명, 페이지 수, 형식(HWP/HWPX)
5. 오류/로딩 메시지 개선
   - 엔진 준비 중 / 문서 분석 중 / 첫 페이지 렌더링 중 등 단계 표시

## 2차 목표
1. 문서 내 찾기
   - 검색어 입력
   - 결과 개수 표시
   - 다음/이전 결과 이동
2. 빠른 페이지 이동 개선
   - 페이지 점프 UX 개선
   - 최근 이동 페이지 기억
3. 보기 모드 개선
   - 화면 맞춤 / 너비 맞춤 / 100%

## 3차 목표
1. 목차/개요 패널
2. 페이지 썸네일
3. 텍스트 선택/복사 가능성 검토
4. 공유로 열기/PWA 재도입 여부 검토

## 기본 원칙
- 폰에서 쓰기 쉬워야 함
- 작은 파일부터 확실히 열리는 게 우선
- 고급 기능보다 안정성을 먼저 확보
- 엔진 한계가 있으면 숨기지 말고 오류를 분명히 보여주기

## 바로 시작할 기본 작업 순서
1. 초기 진입 안정화 검증
2. 확대/축소 버튼 추가
3. 문서 정보 표시
4. 로딩/오류 메시지 개선
5. 그 다음 찾기 기능 가능 여부 점검
