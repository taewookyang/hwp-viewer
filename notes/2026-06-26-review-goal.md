# HWP Viewer 리뷰용 목표 문서

## 한 줄 목표
안드로이드에서 **링크만 열면 바로 쓰기 시작할 수 있는**, 가볍고 안정적인 HWP/HWPX 뷰어를 만든다.

## 제품 목표
- 사용자는 앱 설치 없이 링크만으로 뷰어에 들어올 수 있어야 한다.
- 사용자는 자기 폰에 있는 HWP/HWPX 파일을 바로 열 수 있어야 한다.
- 작은 문서는 확실히 열리고, 실패하면 이유를 이해할 수 있어야 한다.
- 복잡한 고급 편집기보다 **읽기 쉬운 뷰어**에 집중한다.

## 핵심 사용자 경험
1. 링크 접속
2. 파일 열기
3. 문서 확인
4. 페이지 이동 / 확대 / 기본 탐색
5. 필요하면 찾기 기능으로 문장 검색

## 이번 단계 범위
### 포함
- 초기 진입 안정화
- 문서 열기 안정화
- 확대/축소 버튼
- 문서 정보 표시
- 로딩/오류 메시지 개선

### 제외
- 편집 기능
- 클라우드 저장
- 계정 시스템
- 협업 기능
- 완전한 데스크톱급 문서 호환

## 1차 목표
### 1. 초기 진입 안정화
- 링크로 들어갔을 때 무한 로딩이 없어야 한다.
- 파일 선택 전에는 버튼이 항상 눌려야 한다.
- 캐시/서비스워커 때문에 옛 화면이 남지 않게 한다.

### 2. 문서 열기 안정화
- 작은 HWP/HWPX 문서는 우선적으로 안정적으로 열려야 한다.
- 열 수 없는 문서는 멈추지 말고 실패 메시지를 보여줘야 한다.
- 파일 크기/형식/지원 한계를 사용자에게 분명히 알려야 한다.

### 3. 확대/축소
- 핀치 없이도 확대/축소 가능해야 한다.
- 최소한 `축소`, `원래 크기`, `확대` 3가지는 제공한다.
- 모바일 한 손 사용을 해치지 않는 위치에 둔다.

### 4. 문서 정보
- 현재 파일명
- 페이지 수
- 형식(HWP/HWPX)
- 필요 시 파일 크기

### 5. 로딩/오류 메시지 개선
- 엔진 준비 중
- 문서 읽는 중
- 문서 분석 중
- 첫 페이지 렌더링 중
- 지원되지 않거나 손상된 문서일 수 있음

## 2차 목표
### 1. 문서 내 찾기
- 검색어 입력
- 결과 수 표시
- 다음/이전 이동
- 가능하면 현재 페이지 중심으로 탐색

### 2. 빠른 이동 개선
- 페이지 점프 UX 정리
- 자주 가는 페이지 이동 편의 개선

### 3. 보기 모드 개선
- 화면 맞춤
- 너비 맞춤
- 100%

## 3차 목표
- 목차/개요 패널
- 썸네일 목록
- 텍스트 선택/복사 가능성 검토
- 공유로 열기 재검토
- PWA 재도입 여부 재검토

## 성공 기준
- 링크 접속 직후 `파일 열기` 버튼이 항상 눌린다.
- 작은 테스트 문서가 실제 안드로이드에서 열린다.
- 문서가 실패하더라도 앱이 멈추지 않는다.
- 확대/축소와 페이지 이동이 기본적으로 usable 하다.
- 사용자 입장에서 “일단 읽는 용도”로는 충분하다고 느낀다.

## 현재 리스크
- `rhwp.js` / `rhwp_bg.wasm` 엔진 자체 한계
- 일부 HWPX 문서 호환성 문제
- 복잡한 문서에서 모바일 성능 저하 가능성
- 텍스트 검색/하이라이트는 엔진 지원 수준에 따라 구현 난이도가 달라질 수 있음

## 추천 구현 순서
1. 초기 진입 안정화 마무리 검증
2. 확대/축소 버튼 추가
3. 문서 정보 표시
4. 로딩/오류 메시지 개선
5. 찾기 기능 가능성 조사
6. 찾기 기능 설계/구현 여부 결정
