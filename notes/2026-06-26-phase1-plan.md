# HWP Viewer 1차 구현 실행계획

## 목표
1차 목표 범위 안에서 **실사용 체감이 큰 개선**부터 빠르게 적용한다.

## 이번 구현 범위
1. 확대/축소 버튼 추가
2. 문서 정보 표시
3. 로딩/오류 메시지 개선
4. 찾기 기능 가능성 조사 준비

## 작업 순서
### 1. 확대/축소 UI 추가
- 상단 또는 하단에 `- / 100% / +` 형태 버튼 추가
- 모바일에서 한 손으로 누르기 쉬운 위치 배치
- 문서 SVG에 CSS transform 또는 적절한 확대 방식 적용
- 최소/기본/최대 배율 범위 정하기

### 2. 확대 상태 관리
- 현재 배율 상태 저장
- 문서 다시 열 때 기본 배율로 초기화
- 페이지 이동 시 배율 유지 여부 결정
  - 기본: 문서 내에서는 유지

### 3. 문서 정보 표시
- 파일명
- 페이지 수
- 형식(HWP/HWPX)
- 가능하면 파일 크기

### 4. 로딩/오류 메시지 정리
- 엔진 준비 중
- 문서 읽는 중
- 문서 분석 중
- 첫 페이지 렌더링 중
- 실패 원인별 메시지 분리
  - 형식 오류
  - 너무 큰 파일
  - 지원되지 않는 문서
  - 렌더링 실패

### 5. 기본 검증
- 링크 진입 직후 버튼이 눌리는지
- 작은 HWP 파일 열리는지
- 작은 HWPX 파일 열리는지
- 확대/축소 버튼 동작하는지
- 실패 시 앱이 멈추지 않는지

### 6. 찾기 기능 조사 준비
- `rhwp.js`에서 텍스트 추출 API 유무 확인
- 페이지별 텍스트/좌표 추출 가능성 확인
- 가능/불가/우회안 메모 남기기

## 성공 기준
- 초기 진입 화면에서 파일 열기 버튼이 정상 동작
- 문서 열기 후 확대/축소 버튼이 실제로 동작
- 파일 정보가 사용자에게 보임
- 오류가 나도 무한 로딩 없이 빠져나옴
- 찾기 기능 구현 가능성에 대해 근거 있는 판단 가능

## 구현 후 바로 볼 것
- 삼성인터넷 / 크롬 둘 다 진입 문제 없는지
- HWP와 HWPX 중 어느 쪽이 더 불안정한지
- 확대 방식이 글자 선명도를 너무 해치지 않는지
